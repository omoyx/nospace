# AGENTS.md

目标先行，规则少而硬：

- 先读 `@TASK.md`；长期目标只写在那里，不写短期流水账。
- 按 `@task/` 拆子任务并逐个闭环；过程资料、验证记录、资产放到对应 `@task/<id>/`。
- 常用文档放 `@docs/`，可复用脚本放 `@scripts/`，文件名要能说明用途。遇到项目级问题优先查看这两处是否有能提供帮助的内容。
- 遇到会复发的问题，写进 `@mistake/`；以后先查再动手。
- 不提交密钥、令牌、密码、下载包、临时产物和个人隐私数据。
- 需要隔离大量探索上下文时再用 subagents；主上下文保持短、干净、可执行。
- 每个可验证里程碑都要验证、记录结果、`git status --short`，然后主动提交。
- 关键修改完成后必须上线到正式发布环境，并用公开线上地址验证无误；本地运行或本地截图不能替代上线验证。

完成定义：

- 子任务有明确目标、验证命令和结果。
- 相关问题已写入 `@mistake/` 或确认无需新增。
- 文档、脚本或代码变更已通过对应验证。
- 里程碑已提交 git commit。
- 关键修改已完成线上发布，并记录公开地址的验证结果。
